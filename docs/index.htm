Welcome to your new Help Builder Project. You're ready to start creating content for your documentation.

Getting Started

There are a few basic operations you're likely to use repeatedly:

  • Edit any topic by clicking Edit View in the toolbar
  • Add new topics in the tree by clicking New Topic (Ctrl-n) in the toolbar
  • Use the text editor to edit your content using Markdown formatting
  • You can toggle between Edit View and HTML Preview (Ctrl-Tab)
  • To edit content and see a preview click Web Browser Preview (Alt-v)
  • Use Insert Bookmark (Alt-o) to insert links to other topics
  • Use Insert Link (Alt-k) to insert external links
  • Use Insert Image to insert local or Web images into the page
  • Use Screen Capture to capture screen shots to images
  • Build HTML and/or Html Help Output by clicking Build Output in the toolbar
  • Upload your Documentation to a Web site via FTP
  • Build HTML Help, Word or PDF output

Markdown

The editor uses Markdown formatting for editing topics by default. Markdown is a simple text markup format that uses common typographic characters to indicate formatting that is translated to HTML. The format is very easy to learn and very efficient for typing and keeping your hands on the keyboard. If you need help, the toolbar provides helpers for creating marked up text for you. This document you are looking at is written with Markdown.

For more information on the Markdown edit format, please see the following guides:

  • Markdown Editing Basics
  • GitHub Markdown Basics
  • Official Markdown Syntax Guide

Get Busy!

Start by editing this index topic that is your project home page and replace it with your own content (you can't delete this topic). Slice and dice and update this topic and experiment a bit with Markdown if it's new to you. Use the toolbar to help with your formatting and get familiar with the Markdown syntax.

Then add new topics and optionally create a topic hierarchy. As you edit, Help Builder gets out of your way and lets you focus on what you do best: Building quality content.

Finally let Help Builder turn that content into professional output into HTML using Help Builder's built in HTML based templates. Customize the templates if you need to, or use them as is. Output generated can be an HTML Web site, an HTML Help File or a Word or PDF document.

Creating documentation and reference has never been this easy and fun. So, what are you waiting for? Get busy and get it done!
